    משתמשים (Users)

    סוג פעולה כתובת URL פרמטרים בנתיב גוף הבקשה (Body) מה עושה? מי יכול להשתמש? איזה סוגי שגיאות עלולות לחזור?
    POST /api/user/register - userName, email, password רישום משתמש חדש (מחזיר Token) כולם
  • חסרים פרטים (userName, email, password)
  • אימייל כבר קיים (409)
    		•
    POST /api/user/login - email או userName, password התחברות משתמש (מחזיר Token) כולם
  • חסרים פרטים (email/username ו-password)
  • משתמש לא נמצא (401)
  • סיסמה לא נכונה (401)
    		•
    GET /api/user - - מחזירה את כל המשתמשים מנהל בלבד

    הזמנות (Orders)

    סוג פעולה כתובת URL פרמטרים בנתיב גוף הבקשה (Body) מה עושה? מי יכול להשתמש? איזה סוגי שגיאות עלולות לחזור?
    GET /api/order - - מחזירה את כל ההזמנות של המשתמש המחובר משתמש מחובר
  • נדרש אימות
    		•
    GET /api/order/all - - מחזירה את כל ההזמנות (מנהל בלבד) מנהל בלבד
  • נדרש אימות מנהל
    		•
    POST /api/order - address, orderdProducts: [{ productId, name, price, quantity }] מוסיפה הזמנה חדשה משתמש מחובר
  • חסרים פרטים (address, orderdProducts)
  • orderdProducts ריק או לא קיים
  • נדרש אימות
    		•
    PUT /api/order/:id id - מעדכנת סטטוס הזמנה ל-"יצאה לדרך" (isShipped: true) בעלים של ההזמנה או מנהל
  • ID אינו תקין
  • ID אינו קיים (404)
  • נדרש אימות - בעלים של ההזמנה או מנהל
    		•
    DELETE /api/order/:id id - מוחקת הזמנה (רק אם לא נשלחה) מנהל בלבד
  • ID אינו תקין
  • ID אינו קיים (404)
  • אם ההזמנה כבר נשלחה - לא ניתן למחוק (400)
